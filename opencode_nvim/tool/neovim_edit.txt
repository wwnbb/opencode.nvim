Exact string replacement in existing files.

cRITICAL!!!!:
- User may edit during review. Final disk content wins. Re-read after review.
- Treat user review edits as intentional, not mistakes. Do not "fix" or revert them unless user explicitly asks.
- Never auto-retry `rejected`/`partial`/user-edited result unless user asks.

Usage:
- Must use `Read` once before edit, or tool errors.
- From `Read` output, copy content after line-number prefix only. Keep exact tabs/spaces. Never include line numbers.
- `newString` is literal replacement text. Preserve indentation explicitly; if replacing `    foo()`, write `    bar()`, not `bar()`.
- Replacements that remove leading indentation are rejected by default. Set `allowIndentChange=true` only when intentionally dedenting code.
- Prefer existing files. Do not create files unless required.
- `oldString` must match once. Missing => `oldString not found in content`. Multiple => add context or use `replaceAll`.
- Use `replaceAll` for file-wide rename/replacement.
- One replacement per call. For multiple edits, prefer `neovim_apply_patch`.
- Output/metadata describe final disk state: `applied`, `rejected`, or `partial`; trust it.
